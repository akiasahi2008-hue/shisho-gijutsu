# 金史和訳テキスト HTML変換プロトコル

このドキュメントは、テキスト形式の『金史』和訳を、高級感のある読書用HTMLフォーマットへ変換するための標準手順書です。デザインおよびレイアウト基盤は『周書』和訳プロジェクトの仕様に準拠します。

---

## 1. プロジェクトの基本方針

- **対象**: `金史/和訳` フォルダ内のすべての `.txt` ファイル。
- **ゴール**: ブラウザで読みやすく、歴史書としての重厚感（和紙・羊皮紙風）を備えたレスポンシブHTMLへの変換。
- **構成**: CSSはメンテナンス性を考慮しつつ、各HTML内に直接埋め込む（ポータブル形式）。

## 2. 言語処理ルール

### A. 改行の完全再現
- **原則**: **テキストファイルの改行を100%尊重する。**
- **処理**: テキストファイル内の各行を一つの段落（`<p>`）として出力。プログラムによる自動的な行結合は行わない。

### B. インテリジェント・ルビ (Rubi)
特定の難読語や人名・官職名に対し、`<ruby>` タグを自動付与します。
- **2パス処理による自動抽出**: テキスト内の `漢字（ひらがな）` 形式から読みを自動抽出し、そのファイル内の初出箇所にルビを適用する。
- **拡張辞書の活用**: 常用漢字外の難読人名、古代地名、特殊な歴史用語・官職名をプリセット辞書でカバーする。
- **初出ルール**: 各ファイル内でその単語が最初に出現した箇所にのみルビを振る。
- **青空形式の優先**: `｜漢字《ひらがな》` 形式がある場合は、最優先で `<ruby>` に変換する。

### C. 括弧書き（注釈）の扱い
- **重複防止**: ひらがなのみの括弧書き（読み）は、ルビ化と同時に本文から削除して、二重表記を防止する。
- **注釈の保持**: 漢字を含む括弧書き（例：`太宗（李世民）`）は、歴史的注釈と判断し、削除せず本文と同じフォント、太字（`bold`）で表示する。

## 3. デザイン仕様 (CSS)

- **配色**: 
    - **外部背景**: 金茶色（`#c0a154`）
    - **本文領域（Container）**: 和紙色（`#fdfcf0`）
    - **目録ボタン**: 金茶ベース（`#e6d5a7` / Hover: `#d4bc7d`）
- **タイポグラフィ**: 
    - **フォント**: 「Hiragino Mincho ProN」を優先、明朝体（`serif`）。
    - **太さ**: 本文全体（`.container`）を `font-weight: bold;` で表示。
    - **ルビ**: `ruby rt` は `font-weight: normal !important;` とし、読みやすさを確保。
    - **行間**: `line-height: 1.8;` でゆとりを持たせる。
- **レイアウト**: 
    - **中央配置**: `.container` の `max-width` は `900px` とし、左右に十分な余白を設定。
    - **装飾**: 容器に `box-shadow: 0 4px 15px rgba(0,0,0,0.3);` を適用し、立体感を出す。
