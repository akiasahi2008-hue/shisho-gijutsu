# テキスト整備手順（ウィキ残骸・マークアップ除去）

ウィキソース由来の史書テキスト（本プロジェクトでは **金史** の**和訳／原文フォルダ内の `.txt`**）を整える際の手順です。  
**`temp_chunks_deepseek` などのチャンク作業用フォルダは対象外**とします（必要なら別途手で適用してください）。

---

## 0. Git で現状を保存する（推奨）

手順 2〜4 は **和訳・原文の `.txt` を一括で書き換え**ます。実行前に **Git でコミット**しておくと、差分の確認や巻き戻しが容易です。

### 0.1 初回（まだリポジトリがない場合）

プロジェクトのルート（この `テキスト整備手順.md` があるフォルダ）で実行します。パスは環境に合わせて読み替えてください。

```bash
cd /d "E:\マイドライブ\史書\金史"
git init
git config user.name "表示名"
git config user.email "you@example.com"
```

初回コミット前に、追跡から外したいものがあれば **`.gitignore`** を置きます。例:

```gitignore
temp_chunks_deepseek/
__pycache__/
*.py[cod]
```

### 0.2 整備の流れ（コミットの切り方の例）

1. **整備前**: 現状をすべてコミットする。

```bash
git add -A
git status
git commit -m "chore: テキスト整備前の現状保存"
```

2. **手順 2〜4** を順に実行する（各段階ごとにコミットしてもよい）。

```bash
python strip_wiki_templates.py
python remove_toc_lines.py
python remove_wiki_html_artifacts.py
```

3. **整備後**: 変更をコミットする。

```bash
git add -A
git commit -m "chore: ウィキ・HTML整備（手順2〜4）"
```

スクリプトだけ先に追加したい場合は、「スクリプト追加」と「テキスト一括置換」を **別コミット**に分けると履歴が追いやすくなります。

### 0.3 巻き戻しの例

| 目的 | コマンド例（Git 2.23+） |
|------|-------------------------|
| 未コミットの変更をすべて捨てる | `git restore .` |
| 特定ファイルだけ戻す | `git restore -- 和訳/巻001.txt` |
| 直前のコミット状態に戻す（注意: 以降のコミットは残る） | `git revert <コミットハッシュ>` など方針に応じて選択 |

`git log --oneline` でコミット一覧を確認してから操作すると安全です。

## 1. 英字・英単語の混入チェック（任意）

ラテン文字が本文に紛れ込んでいないか確認します。

### 1.1 検索例（正規表現）

対象フォルダを `PATH1`、`PATH2` に読み替えてください。

- いずれかの英字を含む行:
  - パターン: `[A-Za-z]`
  - 対象: `巻*.txt` など

### 1.2 典型の混入

| 種類 | 例 | 対応 |
|------|-----|------|
| MediaWiki 目次 | `__TOC__` `__FORCETOC__` | 手順 3・4 で削除 |
| HTML／transclusion | `<onlyinclude>` `</onlyinclude>` `<poem>` `</poem>` `<blockquote>` | 手順 4 で除去 |
| 独自タグ | `{{YL|…}}` `{{PUA|…}}` `{{*|…}}` | 手順 2 でラッパー除去 |
| フッタ系テンプレ | `{{footer}}` `{{五代作品}}` 等 | 手順 4 で除去 |
| 通常の英語 | `http` `the` など | 個別に判断・修正 |

タグは**意図的なマークアップ**であることが多いです。英字がタグ名部分にだけある場合は、手順 2 でラッパーを外せば本文から英字が消えます。

## 2. マークアップのラッパー除去（中身は残す）

次の **外側だけ** を取り除き、内側を本文に残します（種類により第1フィールドのみ等。詳細は `strip_wiki_templates.py` 先頭コメント）。

- `{{YL|…}}` `{{PUA|…}}` `{{*|…}}` — 従来どおり（`|`～`}}` をそのまま残す）
- `{{!|字形|説明}}` — **第1フィールド（字形）のみ**残す
- `{{--|…}}` `{{-|…}}` — 譜系・注記ラベル（内側全文）
- `{{udots|…}}` `{{ProperNoun|…}}` `{{?|…}}` — 内側全文
- `{{?}}` — `〓` に置換

例: `{{YL|天祐三年}}` → `天祐三年`

### 2.1 注意（入れ子）

`{{*|（…《少帝紀》：{{YL|開運三年…）}}` のように、**注の中に別タグが誤って入っている**箇所があります。  
一度だけ置換すると `{{YL|` が本文に残ることがあるため、**変化がなくなるまで同じ処理を繰り返す**必要があります。

### 2.2 実行用スクリプト

**このフォルダには `strip_wiki_templates.py` を同梱しています。** 既定は `和訳` と `原文` の両方。**原文だけ**処理する場合は `--folders 原文` を付けます。

```bash
python strip_wiki_templates.py
python strip_wiki_templates.py --folders 原文
```

別場所にコピーする場合は、下記を `strip_wiki_templates.py` として保存しても同じです。

```python
# -*- coding: utf-8 -*-
"""{{YL|…}} {{PUA|…}} {{*|…}} のラッパーを除去し、内側の文字だけ残す。"""
from __future__ import annotations

from pathlib import Path

# ▼ 編集: テキストのルートと、処理するサブフォルダ名
ROOT = Path(__file__).resolve().parent
FOLDERS = ("和訳", "原文")

PREFIXES = ("{{YL|", "{{PUA|", "{{*|")


def strip_templates(s: str) -> str:
    out: list[str] = []
    i = 0
    n = len(s)
    while i < n:
        best = -1
        which = ""
        for p in PREFIXES:
            j = s.find(p, i)
            if j != -1 and (best == -1 or j < best):
                best = j
                which = p
        if best == -1:
            out.append(s[i:])
            break
        out.append(s[i:best])
        start_inner = best + len(which)
        end = s.find("}}", start_inner)
        if end == -1:
            out.append(s[best:])
            break
        out.append(s[start_inner:end])
        i = end + 2
    return "".join(out)


def main() -> None:
    changed_files = 0
    for name in FOLDERS:
        d = ROOT / name
        if not d.is_dir():
            print(f"skip (not found): {d}")
            continue
        for path in sorted(d.glob("*.txt")):
            raw = path.read_text(encoding="utf-8")
            text = raw
            while True:
                new = strip_templates(text)
                if new == text:
                    break
                text = new
            if text != raw:
                path.write_text(text, encoding="utf-8", newline="")
                changed_files += 1
                print(path)
    print(f"done. updated {changed_files} files.")


if __name__ == "__main__":
    main()
```

**別プロジェクトで使う場合:** `FOLDERS` を実際のフォルダ名に変更するか、`main()` 内の `d.glob("*.txt")` を `d.rglob("*.txt")` などに変えて再帰的に掛けるようにしてください。

## 3. `__TOC__` の削除

MediaWiki の目次プレースホルダ行を削除します。

### 3.1 実行用スクリプト

**`remove_toc_lines.py`** を同梱しています。

```bash
python remove_toc_lines.py
python remove_toc_lines.py --folders 原文
```

### 3.2 ワンライナー（参考）

`ROOT` とフォルダ名を読み替えて実行してください。

```python
from pathlib import Path

ROOT = Path(r"e:/マイドライブ/史書/金史")  # 編集
for folder in ("和訳", "原文"):  # 編集
    d = ROOT / folder
    for p in d.glob("*.txt"):
        s = p.read_text(encoding="utf-8")
        if "__TOC__" not in s:
            continue
        new = s.replace("__TOC__\n", "").replace("__TOC__\r\n", "")
        if new == s:
            new = s.replace("__TOC__", "")
        p.write_text(new, encoding="utf-8", newline="")
        print(p.name)
```

改行コードが混在する環境では、`splitlines()` で行ごとに `__TOC__` だけの行を捨てる方法でも構いません。

## 4. ウィキ／HTML残骸の除去

Wikisource 由来のテキストに、**本文ではない英字・タグ**が残ることがあります。典型例は次のとおりです。

| 種類 | 例 | 処理 |
|------|-----|------|
| 強制目次 | `__FORCETOC__` | 該当行削除、または行内から除去 |
| transclusion | `<onlyinclude>` 単独行、`</onlyinclude>{{footer}}…` 行 | 行削除 |
| ブロックタグ | `<poem>` `</poem>` `<blockquote>` `</blockquote>` | **タグだけ**除去（中身の漢文は残す） |
| フッタ等 | `{{footer}}` `{{五代作品}}` など | 文字列として除去 |
| 下線・欠字テンプレ | `{{ul|…}}`（内側の文字を残す） `{{SKchar|…}}`（`〓` に置換） | 手順 4 |
| 表組スタイル行頭 | `style=color:#…\|` のような行頭（`|` 以降が本文） | 手順 4 で行頭除去 |
| wiki/HTML装飾 | `'''…'''`（太字記号） `<br>` 系 | 手順 4 で除去 |

`<poem>` で囲まれた書誌リストなどは、タグを外すと普通の行として残ります。

### 4.1 実行用スクリプト

**`remove_wiki_html_artifacts.py`** を同梱しています。

```bash
python remove_wiki_html_artifacts.py
python remove_wiki_html_artifacts.py --folders 原文
```

**実行前に Git でコミット**しておくと、差分の確認・巻き戻しが容易です（手順 **0** 参照）。

### 4.2 注意

- **`temp_chunks_deepseek` は対象外**（手順 2 と同様）。
- 本文に偶然 `</poem>` と同一の並びが含まれる可能性は極めて低いですが、万一あれば当該箇所を手で確認してください。

## 6. 作業の推奨順序

1. **Git** — 詳細は **手順 0**。未使用の場合はフォルダのバックアップでも可。
2. **手順 2**（マークアップラッパー除去）— 入れ子対策の **while ループ必須**
3. **手順 3**（`__TOC__` 削除）
4. **手順 4**（ウィキ／HTML残骸除去）
5. **手順 1**（任意・再チェック）で `[A-Za-z]` が残っていないか確認

## 7. このフォルダ（金史）について

- ルートに `strip_wiki_templates.py`・`remove_toc_lines.py`・`remove_wiki_html_artifacts.py` を同梱しています。いずれも既定は `--folders 和訳,原文`（カンマ区切り）です。
- 初回クローン直後は、手順 2〜4 は**未実行**の想定です。実行前に **手順 0**（Git またはフォルダのバックアップ）を推奨します。
- 金史の和訳に五代史専用の `{{五代作品}}` 等が現れない場合でも、手順 4 の除去リストに含まれていても害はありません。金史側で多い別テンプレートが分かったら、`remove_wiki_html_artifacts.py` の `_SUBSTRIP` / `_EXTRA_BRACE` に追記してください。

## 8. 原文優先パイプライン（金史・推奨作業順）

和訳に入る前に **原文だけ** を揃える流れです（`workflow.py` 単体の fetch → split とは順序が異なります）。

1. **索引の URL 一覧**  
   `python regen_url_list.py`（必要時）

2. **一括取得＋表のある巻の除外**  
   `python bulk_preflight_fetch.py --base-dir "E:\マイドライブ\史書\金史"`  
   - 各巻の **wikitext raw** に `{|`（ウィキ表）または `<table`（HTML 表）が含まれる場合、その巻は **原文を保存せず**、**`URLリスト.txt` から該当行を削除**します（和訳対象外）。  
   - ログ: `表検出_除外巻.txt`  
   - 動作確認: `--limit 3 --dry-run`（このとき URLリストは書き換えません）

3. **（任意）既に原文だけ先に揃えていた場合の後からの表検出**  
   `python purge_volumes_with_tables.py`  
   - `原文/` を走査し、表ありならファイル削除＋ URLリストから削除。

4. **原文のテキスト整備（手順 2〜4 相当）**  
   まとめて実行する場合:  
   `python cleanup_raw_pipeline.py`  
   （内部で `strip_wiki_templates` / `remove_toc_lines` / `remove_wiki_html_artifacts` を **`--folders 原文`** で実行し、最後に `scan_raw_text_issues.py` でレポート）

5. **文字化け・英字の確認**  
   `cleanup_raw_pipeline` 実行後、`原文_テキスト調査.txt` を確認し、**置換文字（U+FFFD）や英字行**を手修正（または別途スクリプト）。自動削除は行いません。

6. **和訳パイプライン**  
   原文が整った巻だけ、`workflow.py` または `run_translate.bat`（いずれも既定で 原文/ の TXT のみ使用。Wikisource から取るときは `workflow.py` に `--fetch`）。

## 9. トラブル時

- **`}}` が本文中にあり、誤って短く切れる**  
  → 現状のスクリプトは「最初の `}}`」で閉じるため、極めて稀に誤動作する可能性があります。その場合は当該ファイルだけ手修正するか、タグ形式をデータで確認してください。
- **UTF-8 以外のエンコーディング**  
  → `read_text` / `write_text` の `encoding` を合わせてください。
- **手順 4 で意図せずタグが消えた**  
  → `remove_wiki_html_artifacts.py` の `_SUBSTRIP` ／ `_LINE_REMOVE_RES` を当該プロジェクト向けに調整するか、当該巻のみ手修正してください。
